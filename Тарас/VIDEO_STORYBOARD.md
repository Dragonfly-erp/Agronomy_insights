# Сценарій відео по кейсу

1. Показати, що SMS відкриває карту некоректно.
2. Пояснити: таблиця DBF може бути нормальною, але геометрія SHP може бути зламана.
3. Пояснити координатний дефект без публікації реальних координат.
4. Пояснити, чому ширина жатки важлива, але не була головною причиною поломки масштабу.
5. Показати, що другий файл був підмножиною першого, а не повним другим комбайном.
6. Пояснити провали: якщо сирих проходів немає, їх можна тільки інтерполювати.
7. Показати фінальну нормалізовану карту `YLD_NORM`.
8. Дати чеклист для механізатора: поле, культура, задача, ширина жатки, датчик жатки, area counter, GPS, калібрування, повний експорт.
